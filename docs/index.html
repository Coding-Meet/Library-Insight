The problem

AI should code against the library version you actually use.

AI assistants often learn from old docs, latest web examples, or a different library version than your project has installed. That creates wrong methods, deprecated code, and broken implementations.

Without version context

AI guesses from docs and web examples

  • ✕ AI may use APIs from the latest version, not your installed version
  • ✕ It may suggest methods that do not exist in your dependency
  • ✕ Official docs can be incomplete, old, or not updated with releases
  • ✕ Deprecated methods get reused instead of the recommended replacement
AI: use library example from the web Error: method not found in this version
With Library Insight

AI gets exact APIs from your library

  • ✓ Scans the exact JAR/AAR or Maven version in your project
  • ✓ Shows real classes, methods, signatures, and Kotlin metadata
  • ✓ Helps AI avoid unavailable or deprecated APIs
  • ✓ Compares versions so upgrades are easier to understand
library-insight explain Retrofit OK: exact signatures from your version

What it does

Simple commands for real library details.

Give it a Maven coordinate, JAR, AAR, or build folder. It scans the compiled files and turns them into searchable API information.

Finds public APIs

Reads classes, constructors, methods, properties, generics, Kotlin flags, and parameter details from compiled files.

Works with local cache

Checks your Gradle cache first. If Android Studio already downloaded the library, it can scan without downloading again.

Made for AI agents

Creates small class-level files so assistants can read only the exact API they need instead of one huge document.

Compares versions

Shows added, removed, and changed APIs so you can spot breaking changes before upgrading a dependency.

Docs

Start with the right workflow.

Use this flow when AI needs to implement a dependency correctly for the exact version in your project.

01

Install the command

You need JDK 17 or newer. Run the installer script to configure library-insight globally.

bash 
$ curl -fsSL https://raw.githubusercontent.com/Coding-Meet/Library-Insight/main/install.sh | bash
02

Initialize workspace skill

Scopes the agent skill to your current project root, creating .agents/skills/library-insight/SKILL.md for workspace-scoped AI agents.

bash 
$ library-insight init
SUCCESS: AI Agent Skill initialized at .agents/skills/library-insight/SKILL.md
03

Scan and verify before coding

Scan the dependency, search for the class, then explain it before writing implementation code.

bash 
$ library-insight scan com.squareup.retrofit2:retrofit:2.11.0
$ library-insight search Retrofit
$ library-insight explain Retrofit
When adding a library

Scan first

Use the Maven coordinate or local JAR/AAR so the index matches the version installed in your project.

When AI suggests code

Verify the method

Use search and explain to confirm the class and method really exist.

When upgrading

Compare versions

Use diff to catch removed, changed, or deprecated APIs before rewriting code.

Command reference

Every command in one place.

scan

Builds the API index from a Maven coordinate, JAR, AAR, directory, or Gradle output.

library-insight scan com.squareup.retrofit2:retrofit:2.11.0

search

Finds packages, classes, methods, and properties in the scanned index.

library-insight search Retrofit

explain

Prints class details, constructors, methods, properties, signatures, and docs.

library-insight explain Retrofit

diff

Compares two library versions and highlights added, removed, changed, or breaking APIs.

library-insight diff com.squareup.retrofit2:retrofit:2.9.0 com.squareup.retrofit2:retrofit:2.11.0

ai-export

Creates split class-level JSON files for AI agents under build/ai-context/.

library-insight ai-export

export

Exports the scanned index as Markdown or JSON for human-readable docs.

library-insight export markdown

init

Adds the Library Insight agent skill to the current workspace.

library-insight init

skills

Manages workspace agent skill installation.

library-insight skills list

doctor

Checks Java, cache, and AI skill setup.

library-insight doctor

clear-cache

Deletes downloaded Maven artifacts from the local Library Insight cache.

library-insight clear-cache

uninstall

Cleanly removes global binary symlink, installation files, and registered AI agent skills.

curl -fsSL https://raw.githubusercontent.com/Coding-Meet/Library-Insight/main/uninstall.sh | bash
